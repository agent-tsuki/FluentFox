# Code Style & Linting

This document describes the formatting and linting configuration that is already in place. All rules are enforced automatically — the CI pipeline runs `make lint` on every push.

Run before committing:
```bash
make lint   # golangci-lint
make tidy   # go mod tidy + verify
```

---

## Formatting

### `gofmt` — canonical Go formatting

All Go code must be formatted with `gofmt`. There is no negotiation on style — `gofmt` output is the style.

```bash
gofmt -w ./...
# or via the linter:
make lint
```

The linter config enables `simplify: true`, which means `gofmt -s` is applied. This collapses things like `[]T{T{}}` into `[]T{{}}`.

### `goimports` — import ordering

Imports must be in three groups, separated by blank lines:

```go
import (
    // 1. stdlib
    "context"
    "fmt"
    "net/http"

    // 2. third-party
    "github.com/go-chi/chi/v5"
    "github.com/google/uuid"

    // 3. internal (local-prefix: github.com/fluentfox/api)
    "github.com/fluentfox/api/internal/middleware"
    "github.com/fluentfox/api/pkg/response"
)
```

`goimports` enforces this ordering automatically. The local prefix `github.com/fluentfox/api` is configured in `.golangci.yml` so internal packages always sort into their own group.

## Linters

Configured in [.golangci.yml](/.golangci.yml). The linter runs with a 5-minute timeout.

| Linter | What it catches |
|--------|----------------|
| `errcheck` | Unchecked error return values — every error must be handled or explicitly discarded with `_` |
| `govet` | Suspicious code constructs (printf format mismatches, unreachable code, etc.) |
| `staticcheck` | Deeper static analysis — deprecated API usage, impossible conditions, bugs |
| `gosimple` | Suggests simpler constructs where equivalent (e.g., `x == true` → `x`) |
| `unused` | Exported and unexported identifiers that are never referenced |
| `gofmt` | Formatting violations |
| `goimports` | Import ordering violations |
| `misspell` | Common English misspellings in comments, strings, and identifiers |
| `godot` | Comments must end with a period |

### `errcheck` settings

```yaml
check-type-assertions: true   # (x.(T)) must check the ok bool
check-blank: false            # _ = someErr() is allowed
```

Every error return must be handled:
```go
// Bad — linter will flag this
rows, _ := pool.Query(ctx, sql, args...)

// Good
rows, err := pool.Query(ctx, sql, args...)
if err != nil {
    return fmt.Errorf("listUsers: %w", err)
}
```

### `godot` — comment punctuation

All exported comments (and most internal comments) must end with a period. Test files are excluded from this rule.

```go
// Bad
// GetByEmail returns a user by email address

// Good
// GetByEmail returns a user by email address.
func (r *Repository) GetByEmail(...) {}
```

### Exclusions

The linter skips:
- `db/migrations/` — SQL files, not Go
- `db/seeds/` — SQL files, not Go
- `_test.go` files — exempt from `godot` only (all other linters still apply)

## Error Handling

Wrap errors with context using `%w` so the full chain is visible in logs:

```go
// Good — each layer adds its own context
func (r *Repository) GetByID(ctx context.Context, id uuid.UUID) (*User, error) {
    var u User
    err := r.pool.QueryRow(ctx, sqlGetByID, id).Scan(&u.ID, &u.Email)
    if err != nil {
        return nil, fmt.Errorf("userRepo.GetByID: %w", err)
    }
    return &u, nil
}

// Good — service adds its own layer
func (s *Service) GetProfile(ctx context.Context, id uuid.UUID) (*ProfileResponse, error) {
    user, err := s.repo.GetByID(ctx, id)
    if err != nil {
        return nil, fmt.Errorf("userService.GetProfile: %w", err)
    }
    ...
}
```

Domain-specific sentinel errors are defined in `service.go` and checked in `handler.go` using `errors.Is`:

```go
// service.go
var ErrInvalidCredentials = errors.New("invalid credentials")

// handler.go
if errors.Is(err, service.ErrInvalidCredentials) {
    response.Unauthorized(w)
    return
}
```

## Naming Conventions

### Packages

- Package names match the directory name, lowercase, no underscores.
- Test packages use the `_test` suffix: `package auth_test`.

### Types

| Category | Convention | Example |
|----------|-----------|---------|
| DB model | `<Entity>` | `User`, `RefreshToken` |
| Request DTO | `<Action>Request` | `RegisterRequest`, `SubmitReviewRequest` |
| Response DTO | `<Action>Response` | `RegisterResponse`, `LoginResponse` |
| Service errors | `Err<Description>` | `ErrEmailAlreadyExists`, `ErrInvalidCredentials` |

### Functions

| Category | Convention | Example |
|----------|-----------|---------|
| Constructor | `New<Type>` | `NewHandler`, `NewService`, `NewRepository` |
| Handler method | Verb or noun matching the route | `Register`, `Login`, `GetDueCards` |
| Service method | Verb + noun describing the action | `Register`, `SubmitReview`, `GetLeaderboard` |
| Repository method | Verb + noun (DB-oriented) | `Create`, `GetByID`, `ListByUserID`, `Update`, `Delete` |

### Variables

- Acronyms keep their case: `userID` (not `userId`), `dbURL` (not `dbUrl`).
- Context parameter is always named `ctx`.
- Request and response writer always `r` and `w`.

## Comment Style

Comments on exported identifiers are required. They must:
1. Start with the identifier name.
2. End with a period.
3. Be a complete sentence.

```go
// Handler handles HTTP requests for the auth domain.
type Handler struct { ... }

// Register handles POST /auth/register.
func (h *Handler) Register(w http.ResponseWriter, r *http.Request) { ... }
```

For unexported identifiers, comments are encouraged but not required by the linter.

## Test Conventions

Tests live in `internal/<domain>/test/handler_test.go` and use the `<domain>_test` package.

```
internal/auth/test/handler_test.go  →  package auth_test
internal/srs/test/handler_test.go   →  package srs_test
```

**Run tests:**
```bash
make test           # all tests, race detector, isolated DB
make test-coverage  # same + coverage.html report
```

The test database (port 5433, `fluentfox_test`) is ephemeral — it uses `tmpfs` and is torn down after each `make test` run.

Test function naming:
```go
func TestRegister_Success(t *testing.T) { ... }
func TestRegister_DuplicateEmail(t *testing.T) { ... }
func TestLogin_WrongPassword(t *testing.T) { ... }
```

## What the Linter Will Not Catch

These are conventions enforced by code review, not the linter:

- **No global state.** Do not use `init()` functions or package-level variables for mutable state. Dependencies are injected via constructors.
- **No `os.Getenv` outside `config/`.** All configuration flows through `config.Config`.
- **No SQL outside `repository.go`.** Handlers and services must not contain SQL strings.
- **No HTTP types in service or repository.** `http.Request`, `http.ResponseWriter`, and chi imports must not appear in `service.go` or `repository.go`.
- **No business logic in handlers.** Handlers parse, delegate, and respond — nothing more.
